1

访问 supabase.com → 点击 Start your project → 用 GitHub 登录（最快）或邮箱注册
创建新项目
登录后点击 New project → 填写项目名称（如 shein-data）→ 设置数据库密码（记住它）→ 地区选 Southeast Asia (Singapore) → 点击 Create new project

⏳ 等待约 60 秒，项目创建完成
2
建立数据库表
复制 SQL，在 Supabase 控制台运行
进入 SQL Editor
在 Supabase 控制台左侧菜单 → 点击 SQL Editor → 点击 New query
粘贴并运行建表 SQL
复制下面的全部内容，粘贴到 SQL Editor，然后点击 RUN（或按 Ctrl+Enter）：
-- 1. 店铺表 CREATE TABLE IF NOT EXISTS shops ( id TEXT PRIMARY KEY, name TEXT NOT NULL, platform TEXT DEFAULT 'SHEIN', color TEXT DEFAULT '#6366f1', created_at TIMESTAMPTZ DEFAULT NOW() ); -- 2. 销售数据表 CREATE TABLE IF NOT EXISTS sales ( id BIGSERIAL PRIMARY KEY, date DATE NOT NULL, shop_id TEXT NOT NULL REFERENCES shops(id) ON DELETE CASCADE, style_id TEXT NOT NULL, style_name TEXT NOT NULL, orders INT DEFAULT 0, refund_orders INT DEFAULT 0, revenue NUMERIC(12,2) DEFAULT 0, price NUMERIC(10,2) DEFAULT 0, UNIQUE (date, shop_id, style_id) ); CREATE INDEX IF NOT EXISTS idx_sales_date ON sales (date); CREATE INDEX IF NOT EXISTS idx_sales_shop_id ON sales (shop_id); -- 3. 开启读写权限（内部使用） ALTER TABLE shops ENABLE ROW LEVEL SECURITY; ALTER TABLE sales ENABLE ROW LEVEL SECURITY; CREATE POLICY "all_shops" ON shops USING (true) WITH CHECK (true); CREATE POLICY "all_sales" ON sales USING (true) WITH CHECK (true);
✅ 运行成功后会显示 "Success. No rows returned"，这是正常的

常见问题
数据量多少能免费用？
Supabase 免费版给 500MB 数据库空间。你的销售数据每条约 200 字节，500MB 可存约 250万条 记录，几年内都不用担心。
同事打开网址时，会不会很慢？
打开时先显示本地缓存（即时）,同时异步从 Supabase 加载最新数据。整体感觉和本地使用差不多，只有第一次打开时会有 1-2 秒的同步时间。
我在国内，Supabase 能访问吗？
可以，选新加坡节点后国内访问速度不错。API 请求走 HTTPS，不会被屏蔽。
免费版会不会突然收费或停服？
Supabase 是开源项目，免费版稳定存在多年。即使未来政策变化，数据可以随时导出。PostgreSQL 格式，迁移成本极低。
